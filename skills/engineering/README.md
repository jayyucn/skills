# 工程能力
日常编码工作所用到的各项技能。

- **[diagnose](./diagnose/SKILL.md)** — 针对疑难缺陷与性能退化问题开展标准化排查流程：复现问题 → 精简场景 → 提出假设 → 埋点观测 → 修复问题 → 回归测试。
- **[grill-with-docs](./grill-with-docs/SKILL.md)** — 结合现有领域模型对方案进行深度质询、统一术语，并同步更新 `CONTEXT.md` 与 ADR。
- **[triage](./triage/SKILL.md)** — 依照分级处理角色对应的状态流转机制，对问题进行分类处置。
- **[improve-codebase-architecture](./improve-codebase-architecture/SKILL.md)** — 依托 `CONTEXT.md` 中的领域术语以及 `docs/adr/` 中的决策记录，挖掘代码库的架构优化方向。
- **[setup-matt-pocock-skills](./setup-matt-pocock-skills/SKILL.md)** — 为单个代码仓库搭建配套配置（问题跟踪器、分类标签体系、领域文档结构），供其他工程能力调用。
- **[tdd](./tdd/SKILL.md)** — 采用红-绿-重构流程开展测试驱动开发，以完整功能切片为单位完成功能开发或缺陷修复。
- **[to-issues](./to-issues/SKILL.md)** — 基于完整功能切片，将各类方案、规格文档、产品需求文档拆解为可独立认领的 GitHub 任务单。
- **[to-prd](./to-prd/SKILL.md)** — 将当前沟通内容整理为 PRD，并提交为 GitHub 任务单。
- **[zoom-out](./zoom-out/SKILL.md)** — 针对陌生代码模块，切换至宏观视角，梳理整体背景与高层逻辑。
- **[prototype](./prototype/SKILL.md)** — 搭建临时原型以完善设计：可制作可运行的终端应用，验证状态与业务逻辑；也可基于同一路由实现多款风格迥异的界面方案并快速切换对比。

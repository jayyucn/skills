# 技能编写指南
## 流程
1. **收集需求** - 向用户确认以下信息：
   - 该技能适用的任务/业务领域
   - 需要处理的具体使用场景
   - 是否需要可执行脚本，或仅需文字说明
   - 是否需要附带参考资料

2. **编写技能** - 完成以下内容：
   - 编写指令简洁的 SKILL.md 文件
   - 内容超过500行时，补充额外参考文件
   - 存在固定执行操作时，编写工具脚本

3. **与用户核对** - 展示初稿并确认：
   - 是否覆盖全部使用场景
   - 有无内容缺失或表述模糊之处
   - 部分板块是否需要增减细节

## 技能目录结构
```
skill-name/
├── SKILL.md           # 核心指令（必需文件）
├── REFERENCE.md       # 详细文档（按需创建）
├── EXAMPLES.md        # 使用示例（按需创建）
└── scripts/           # 工具脚本（按需创建）
    └── helper.js
```

## SKILL.md 模板
```md
---
name: skill-name
description: 功能简要描述。适用于[具体触发场景]。
---

# 技能名称

## 快速上手
[最简可用示例]

## 执行流程
[复杂任务的分步操作及检查清单]

## 高级功能
[关联独立文件：详见 [REFERENCE.md](REFERENCE.md)]
```

## 描述文案编写要求
智能体选择加载技能时，**仅会读取描述文案**。该内容会和所有已安装技能一同展示在系统提示词中，智能体将根据用户指令，结合描述筛选匹配技能。

**编写目标**：清晰告知智能体以下两点：
1. 该技能具备的功能
2. 触发时机与场景（特定关键词、使用环境、文件类型等）

**格式规范**：
- 字符数上限 1024
- 使用第三人称表述
- 首句说明功能用途
- 第二句固定格式：适用于[具体触发场景]

**优质示例**：
```
从PDF中提取文字与表格、填写表单、合并文档。适用于处理PDF文件，或用户提及PDF、表单、文档提取相关需求时。
```

**劣质示例**：
```
协助处理文档。
```
此类描述无法让智能体区分该技能与其他文档类技能。

## 脚本使用场景
满足以下情况时，需配套工具脚本：
- 操作流程固定（校验、格式整理等）
- 会重复生成相同代码
- 需要针对性处理各类报错

相比临时生成代码，使用脚本可减少资源消耗，并提升运行稳定性。

## 文件拆分规则
出现以下情况时，拆分为多个独立文件：
- SKILL.md 内容超过100行
- 内容分属不同业务领域（如财务、销售数据模型）
- 高级功能使用频率较低

## 自检清单
编写完成后逐项核对：
- [ ] 描述文案包含触发说明（适用于……）
- [ ] SKILL.md 行数不超过100行
- [ ] 不含时效性内容
- [ ] 术语表述统一
- [ ] 附带具体使用示例
- [ ] 引用文件层级不超过一级
